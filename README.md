# SaineX

Десктоп-додаток (Windows / macOS / Linux) для перемикання частоти кадрів
Meta Quest на льоту — кнопкою у вікні програми або затиском кнопки
Oculus/Meta на самому гарнітурі.

## Файли

- `hz_core.py` — логіка підключення по ADB та перемикання Hz (без GUI).
- `hz_gui.py` — вікно програми (tkinter). Запускати саме цей файл.

## Вимоги

1. **Python 3.8+** (на macOS/Linux зазвичай вже встановлений; на Windows —
   завантажити з python.org, при встановленні поставити галочку "Add to PATH").
2. **tkinter** — як правило, вже входить у стандартну збірку Python.
   Якщо на Linux його немає:
   ```
   sudo apt install python3-tk      # Debian/Ubuntu
   sudo dnf install python3-tkinter # Fedora
   ```
3. **ADB (Android Platform Tools)** — завантажити тут:
   https://developer.android.com/tools/releases/platform-tools
   Розпакувати, і додати шлях до папки (де лежить `adb`/`adb.exe`) у PATH.

## Запуск

```
cd sainex
python3 hz_gui.py
```

(на Windows можливо знадобиться `python` замість `python3`)

## Як користуватись

1. Підключи Quest по USB кабелю (дозволь debugging на гарнітурі, якщо спитає).
2. Натисни **Start / Connect**. Програма сама переведе з'єднання на Wi-Fi —
   після цього кабель можна відключити.
3. **Пряме встановлення Hz**: натисни 60 / 72 / 90 / 120 Hz у блоці
   "Display refresh rate" — це міняє саму базову частоту дисплея гарнітури
   (`debug.oculus.refreshRate`). 120 Hz підтримують не всі застосунки.
   Якщо зміна не застосувалась одразу — вимкни і ввімкни екран кнопкою
   живлення на гарнітурі.
4. **Swap interval (дільник кадрів)**: окрема, незалежна функція — обери
   базову частоту (60/72) і пресет, або введи власне значення, і потім:
   - кнопкою **Toggle Hz** у вікні,
   - або затиском кнопки Oculus/Meta на гарнітурі на 0.25–1.5 секунди
     (відпусти кнопку саме в момент появи кола recenter — це і є тригер).
5. **Force Normal** / **Force Target** дозволяють примусово виставити
   конкретний swap-режим, не чекаючи toggle.

### Чим відрізняються ці два механізми

- **Refresh rate (60/72/90/120 кнопки)** — встановлює фактичну частоту
  оновлення екрана гарнітури.
- **Swap interval (toggle/preset)** — не міняє екран, а каже рендереру
  пропускати кадри (наприклад interval=2 на базі 60 Hz ефективно дає 30 fps
  рендеру, хоч екран лишається 60 Hz). Корисно як швидкий "турборежим"
  всередині гри без зміни самого дисплея.

Можна комбінувати: наприклад поставити базу 90 Hz кнопкою, а потім
використовувати toggle для додаткового зниження навантаження під час гри.

### Порада

Щоб тригерити Hz-перемикання було легше під час гри, у налаштуваннях
гарнітури можна перемапити іншу кнопку (наприклад A) на кнопку Oculus —
це зручніше, ніж використовувати саму системну кнопку меню.

### Прив'язка тригера до іншої кнопки (не Oculus/Meta)

За замовчуванням тригер — кнопка Oculus/Meta (код `009f`). Якщо хочеш
прив'язати іншу кнопку контролера (наприклад B/Y):

1. Підключись (Start / Connect).
2. Натисни **Detect...** в блоці "Trigger button" — програма почне
   слухати 6 секунд.
3. Поки йде прослуховування, кілька разів **затисни і відпусти** потрібну
   кнопку на контролері (саме ту, яку хочеш зробити тригером — не чіпай
   інші кнопки в цей час).
4. Програма покаже знайдений код і сама підставить його в поле
   "Button code". Якщо вона побачила декілька різних кодів — натисни
   Detect ще раз і тримай лише одну кнопку.
5. Тепер замість Oculus/Meta-кнопки тригером буде обрана тобою кнопка
   (тримай 0.25–1.5 сек, як і раніше).

**Примітка:** не всі кнопки фізичних контролерів Quest (наприклад A/B/X/Y
на Touch-контролерах) обов'язково проходять через `getevent` так само, як
системна кнопка Oculus — це залежить від того, як ОС маршрутизує
Bluetooth-події контролера. Якщо Detect не бачить нічого при натисканні
кнопки контролера, спробуй ще раз притримати довше (0.5–1с), і якщо й це
не допомагає — ймовірно ця конкретна кнопка не публікується в той самий
`/dev/input` потік, і прив'язати її таким способом не вийде.

## Збірка в портативний .exe (Windows)

Це пакує `adb.exe` та потрібні DLL прямо всередину одного файлу — не треба
тримати поруч папку `platform-tools`.

1. Встанови PyInstaller (одноразово):
   ```
   pip install pyinstaller
   ```

2. Перейди в командному рядку в папку, де лежать `hz_core.py`, `hz_gui.py`
   та файли з `platform-tools` (`adb.exe`, `AdbWinApi.dll`, `AdbWinUsbApi.dll`):
   ```
   cd C:\Users\saine\Downloads\cheat
   ```

3. Запусти збірку:
   ```
   pyinstaller --onefile --noconsole --name "SaineX" ^
     --add-binary "adb.exe;." ^
     --add-binary "AdbWinApi.dll;." ^
     --add-binary "AdbWinUsbApi.dll;." ^
     hz_gui.py
   ```
   (символ `^` — перенесення рядка в cmd; якщо пишеш в PowerShell, заміни
   `^` на `` ` `` або введи все одним рядком без переносів.)

4. Готовий файл буде тут:
   ```
   dist\SaineX.exe
   ```
   Це повністю портативний exe — можна переносити окремо, без папки
   platform-tools. При запуску він сам розпаковує `adb.exe` у тимчасову
   папку і користується ним.

### Якщо щось пішло не так

- **"adb not found on PATH"** при запуску exe — переконайся, що команда
  збірки виконувалась саме в папці, де лежать `adb.exe` і обидва `.dll`
  файли, і що шляхи в `--add-binary` написані правильно (без друкарських
  помилок у назвах файлів).
- Антивірус/Windows Defender іноді підсвічує exe, зібрані PyInstaller'ом,
  як підозрілі (false positive) — це поширена особливість самого
  PyInstaller, а не цього коду.
- Якщо хочеш консоль для діагностики помилок — прибери `--noconsole`
  з команди й перезбери.

## Примітки

- Не виставляй надто низьку Hz — це просто сповільнить гру/відео.
- Якщо з'єднання обірвалось і не відновлюється самостійно — натисни Stop,
  потім Start ще раз.
